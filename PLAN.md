# LLM API Gateway 仓库计划

## 目标
建立一个面向内网服务的 LLM API 跳板网关，支持：

1. 使用配置文件定义上游服务与鉴权信息，避免把密钥硬编码。
2. 提供统一入口 `/` 下转发请求到目标 LLM（兼容 OpenAI 风格路径）。
3. 对请求方进行内网访问鉴权（token 白名单）。
4. 保留错误码与响应内容，支持流式转发（SSE）与普通响应透传。
5. 可直接本地运行（`uv` 管理），并提供可复用的 Docker 化启动方式。

## 里程碑

- [ ] 里程碑 1：搭建仓库基础结构与依赖，完成配置模型与加载。
- [ ] 里程碑 2：完成鉴权逻辑、路径路由匹配和上游请求转发。
- [ ] 里程碑 3：补齐流式响应、错误处理与日志。
- [ ] 里程碑 4：补充最小化测试与文档（含运行方式、配置示例）。
- [ ] 里程碑 5：提供可直接启动的生产运行方式（systemd/system/docker）。
