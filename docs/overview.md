# 架构说明

本仓库采用“入口网关 + 配置路由 + 上游转发”的最小结构。

- 请求先到 `FastAPI` 网关（`api_gateway.app.create_app`）。
- 经过鉴权校验（`auth.enabled` + `auth.tokens` 白名单）后，按路径前缀匹配到上游。
- 组装请求头并注入上游密钥（`api_key_env`）后，用 `httpx` 进行转发。
- 根据请求体中的 `stream` 字段，按普通响应或 SSE 流式响应透传。

该结构有意保留了最少耦合，便于后续新增：
- 限流/配额；
- JWT/OAuth 鉴权；
- 指标与审计日志；
- 多路径与多集群策略。
